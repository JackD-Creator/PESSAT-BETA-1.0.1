________________________________________
RINGKASAN PRODUK
Livestock Management System adalah aplikasi berbasis web dan mobile yang membantu peternak sapi potong, sapi perah, domba, dan kambing mengelola seluruh operasional peternakan secara digital. Sistem ini mengintegrasikan pencatatan data ternak, pemantauan kesehatan, manajemen pakan, inventaris, keuangan, serta pelaporan dalam satu platform dengan dashboard analitik real-time.
________________________________________


PERSONA PENGGUNA
1.	Pemilik (Owner) - Pemilik usaha peternakan. Kebutuhan: Melihat performa bisnis, profitabilitas, tren jangka panjang, laporan keuangan.
2.	Manajer (Manager) - Penanggung jawab operasional harian. Kebutuhan: Mengelola data ternak, inventaris, keuangan, formulasi pakan, tugas staf.
3.	Pekerja Kandang (Worker) - Staf lapangan (pakan, kesehatan, produksi). Kebutuhan: Mencatat aktivitas harian: pemberian pakan, produksi susu, laporan kesehatan ternak.
________________________________________



DASHBOARD – PUSAT KENDALI & WAWASAN
Dashboard adalah halaman utama yang menampilkan data penting secara real-time dari seluruh modul, membantu pengambilan keputusan berbasis data.
Komponen Dashboard:
1.	Ringkasan Populasi & Kondisi Ternak
Sumber Data: animals (GROUP BY species, status)
Tampilan: Total ternak per spesies (sapi potong, sapi perah, domba, kambing), dikelompokkan per status: sehat, sakit, bunting, laktasi, kering, afkir, mati.
2.	Status Kesehatan & Peringatan Dini
Sumber Data: health_records, vaccinations, alerts, breeding_events
Tampilan: Ringkasan penyakit aktif, vaksinasi mendatang (3 hari), ternak dengan gejala abnormal, notifikasi stok obat menipis, perkiraan kelahiran.
3.	Produktivitas Utama
Sumber Data: daily_production, weight_records
Tampilan: Sapi perah: produksi susu (liter/hari); Sapi potong/domba/kambing: ADG (pertambahan bobot harian), bobot rata-rata; Grafik tren 7 hari terakhir.
4.	Manajemen Pakan & Inventaris
Sumber Data: feed_inventory, feed_consumption, feed_purchases
Tampilan: Stok pakan terkini per jenis, estimasi hari tersisa, peringatan stok di bawah threshold, biaya pakan bulan ini, nilai total inventaris.
5.	Keuangan & Biaya Operasional
Sumber Data: financial_transactions
Tampilan: Ringkasan arus kas (cash in/out), laba rugi bulan berjalan, biaya operasional per kategori (pakan, obat, tenaga kerja, utilitas), biaya per ekor.
6.	Tugas & Aktivitas Mendatang
Sumber Data: tasks
Tampilan: Daftar tugas hari ini dan overdue: pemberian pakan, vaksinasi, inseminasi, kontrol kesehatan, pemerahan.
7.	Peta & Kondisi Lahan
Sumber Data: locations, animals
Tampilan: Tampilan visual lokasi kandang/paddock, status hunian per kandang, kapasitas, rotasi penggembalaan.

CORE FEATURES – FITUR UTAMA
1.	MANAJEMEN INVENTARIS TERNAK
1.1 Registrasi & Identitas Ternak
Setiap ternak didaftarkan dengan data individu lengkap:
Data Umum (Semua Komoditas):
•	Tag ID (eartag/QR)
•	RFID (opsional)
•	Spesies: cattle, sheep, goat
•	Ras/Breed: Sapi Perah (FH, Jersey), Sapi Potong (Brahman, Simmental), Domba (Merino, Garut, Dorper), Kambing (Boer, Ettawa, Kacang)
•	Jenis Kelamin
•	Tanggal Lahir
•	Bobot Lahir (kg)
•	Bobot Terkini (kg)
•	Warna
•	Foto
•	Status Fisiologis: healthy, sick, pregnant, lactating, dry, culled, sold, dead
•	Purpose (Tujuan Pemeliharaan): dairy, beef, wool, dual, breeding
•	Induk (dam_id)
•	Pejantan (sire_id)
•	Lokasi Kandang Saat Ini
•	Cara Perolehan: born, purchased, gift
•	Harga Beli (jika dibeli)
1.2 Atribut Spesifik per Komoditas
Atribut Sapi Perah (Purpose: dairy):
•	lactation_number (Laktasi Ke) - number
•	days_in_milk (Hari Laktasi) - number, hari
•	milk_yield_morning (Susu Pagi) - number, liter
•	milk_yield_evening (Susu Sore) - number, liter
•	total_305d_milk (Total Susu 305 Hari) - number, liter
•	body_condition_score (BCS) - number, 1-5
•	dry_period_start (Tanggal Mulai Kering) - date
Atribut Sapi Potong (Purpose: beef):
•	body_condition_score (BCS) - number, 1-5
•	average_daily_gain (ADG) - number, kg/hari
•	feed_conversion_ratio (FCR) - number
•	target_slaughter_weight (Target Bobot Potong) - number, kg
•	estimated_slaughter_date (Estimasi Tanggal Potong) - date
Atribut Domba:
•	body_condition_score (BCS) - number, 1-5
•	wool_grade (Grade Wol) - text
•	wool_yield_per_shearing (Hasil Wol/Cukur) - number, kg
•	last_shearing_date (Tanggal Cukur Terakhir) - date
•	average_daily_gain (ADG) - number, kg/hari
•	litter_size_avg (Rata-rata Anak) - number, ekor
Atribut Kambing:
•	body_condition_score (BCS) - number, 1-5
•	milk_yield_per_day (Susu/Hari) - number, liter (untuk kambing perah)
•	average_daily_gain (ADG) - number, kg/hari
•	litter_size_avg (Rata-rata Anak) - number, ekor
•	purpose (Tujuan) - text: meat, milk, dual
1.3 Silsilah & Pemuliaan
•	Self-reference: dam_id dan sire_id merujuk ke tabel animals
•	Tabel genetic_records (opsional): koefisien inbreeding, EBV susu/pertumbuhan/reproduksi/wol, DNA sample ID
1.4 Bobot & Pengukuran Berkala
Tabel weight_records mencatat: weigh_date, weight_kg, body_condition_score, chest_girth_cm, body_length_cm, height_cm
Setiap insert otomatis update animals.current_weight_kg dengan nilai terbaru.
1.5 Pemindahan Ternak
Tabel animal_movements mencatat perpindahan antar kandang/paddock dengan tanggal, alasan, petugas.
Otomatis update animals.current_location_id.
1.6 Pengelompokan Ternak
Tabel herd_groups dan herd_group_members untuk manajemen kolektif.
Contoh grup: "Kandang A - Sapi Perah Laktasi", "Paddock 1 - Sapi Potong Penggemukan".
Grup digunakan untuk pemberian pakan massal dan vaksinasi massal.

2.	PEMANTAUAN KESEHATAN & REPRODUKSI
2.1 Rekam Medis (health_records)
Atribut: record_date, type (checkup/illness/injury/treatment/surgery/preventive), diagnosis, treatment, vet_name, cost, is_resolved, follow_up_date.
Trigger: Insert dengan cost > 0 menghasilkan transaksi expense cash_out kategori vet_service.
Insert dengan type 'illness' mengubah status ternak menjadi 'sick'.
Update is_resolved = TRUE mengembalikan status ke 'healthy'.
2.2 Vaksinasi (vaccinations)
Atribut: vaccine_name, batch_number, date_administered, next_due_date, cost, animal_id atau herd_group_id (untuk vaksinasi massal).
Trigger: Insert dengan cost > 0 menghasilkan transaksi expense cash_out kategori vaccination.
next_due_date kurang dari atau sama dengan hari ini + 3 hari menghasilkan alert dan task otomatis.
2.3 Reproduksi (breeding_events)
Event types: heat, insemination, pregnancy_check, birth, abortion, dry_off.
Atribut: sire_id, expected_due_date, actual_birth_date, offspring_count, offspring_tag_ids, cost.
Parameter Reproduksi per Komoditas:
•	Sapi Perah/Potong: Lama Kebuntingan 280 hari, Masa Kosong Ideal 60-90 hari, Umur Kawin Pertama 15-18 bulan, Siklus Birahi 21 hari
•	Domba: Lama Kebuntingan 147 hari, Masa Kosong Ideal 30-45 hari, Umur Kawin Pertama 8-10 bulan, Siklus Birahi 17 hari
•	Kambing: Lama Kebuntingan 150 hari, Masa Kosong Ideal 30-45 hari, Umur Kawin Pertama 8-10 bulan, Siklus Birahi 21 hari
Trigger:
•	Insert cost > 0: transaksi expense cash_out kategori breeding
•	pregnancy_check positif: status berubah menjadi 'pregnant'
•	birth: status berubah menjadi 'lactating', otomatis membuat record anak baru di animals
•	dry_off: status berubah menjadi 'dry'
•	expected_due_date <= hari ini + 7 hari: menghasilkan alert
2.4 Pemakaian Obat (medicine_usages)
Atribut: medicine_id, usage_date, health_record_id (opsional), animal_id, quantity, cost_per_unit_at_time, total_cost.
Trigger: Kurangi medicine_inventory, transaksi expense non_cash kategori medicine_usage.

3.	MANAJEMEN PAKAN & INVENTARIS
3.1 Master Data Pakan (feeds)
Kategori Pakan untuk Ruminansia:
•	Hijauan (Forage): Rumput Segar (Rumput Gajah, King Grass, Setaria), Leguminosa (Lamtoro, Kaliandra, Indigofera), Jerami (Jerami Padi, Jerami Jagung), Silase (Silase Rumput, Silase Jagung)
•	Konsentrat: Sumber Energi (Dedak Padi, Onggok, Molases, Jagung Giling), Sumber Protein (Bungkil Kedelai, Tepung Ikan, DDGS), Pakan Komplit (Konsentrat Komersial)
•	Mineral: Mineral Makro (Garam, Kapur, Tepung Tulang), Mineral Mikro (Premix)
•	Suplemen: Vitamin (Premix A,D,E), Aditif (Probiotik, Enzim)
•	Limbah Pertanian: Ampas Tahu, Ampas Kelapa, Kulit Kopi
Kandungan Nutrisi Lengkap:
•	dry_matter_pct (% Bahan Kering)
•	crude_protein_pct (% Protein Kasar)
•	crude_fiber_pct (% Serat Kasar)
•	crude_fat_pct (% Lemak Kasar)
•	ash_pct (% Abu)
•	tdn_pct (% Total Digestible Nutrients)
•	metabolizable_energy (kcal/kg)
•	calcium_pct (% Ca)
•	phosphorus_pct (% P)
•	shelf_life_days (Umur Simpan)
•	storage_condition (Kondisi Penyimpanan)
3.2 Stok Pakan dengan Nilai (feed_inventory)
Menggunakan metode Moving Average.
Atribut: quantity_on_hand (kg), avg_cost_per_unit (Rp/kg), total_cost (Rp), min_threshold.
Rumus: new_avg = ((qty_lama x avg_lama) + (qty_beli x harga_beli)) / (qty_lama + qty_beli)
Alert: quantity_on_hand < min_threshold menghasilkan alert stok menipis.
3.3 Pembelian Pakan (feed_purchases)
Trigger:
1.	Update feed_inventory: qty +, hitung ulang avg_cost, update total_cost
2.	Insert financial_transactions: expense cash_out kategori feed_purchase
3.4 Pemakaian Pakan (feed_consumption)
Per individu ternak atau per grup.
Trigger:
1.	Ambil avg_cost_per_unit terkini dari feed_inventory
2.	Hitung total_cost = quantity x avg_cost_per_unit
3.	Update feed_inventory: qty -, total_cost = qty x avg_cost
4.	Insert financial_transactions: expense non_cash kategori feed_usage
3.5 Formulasi Ransum (feed_formulas & feed_formula_items)
Untuk peternakan yang mencampur pakan sendiri.
Contoh Formula:
•	Ransum Sapi Perah Laktasi: 50% Hijauan, 30% Konsentrat, 15% Bungkil, 5% Mineral
•	Ransum Penggemukan Sapi Potong: 40% Hijauan, 45% Konsentrat, 10% Onggok, 5% Mineral
•	Ransum Domba Penggemukan: 35% Hijauan, 45% Konsentrat, 15% Bungkil, 5% Mineral
Kalkulasi Otomatis: Protein kasar hasil formulasi, TDN hasil formulasi, Biaya per kg ransum.
3.6 Standar Kebutuhan Nutrisi (nutrition_requirements)
Kebutuhan Nutrisi per Komoditas:
•	Sapi Perah Laktasi Awal: BK 15-18 kg/hari, PK 16-18%, TDN 68-72%, Ca 0.6-0.8%, P 0.4-0.5%
•	Sapi Perah Laktasi Tengah: BK 13-16 kg/hari, PK 14-16%, TDN 65-68%, Ca 0.5-0.7%, P 0.35-0.45%
•	Sapi Perah Kering: BK 8-10 kg/hari, PK 12-13%, TDN 55-60%, Ca 0.4-0.5%, P 0.3-0.35%
•	Sapi Potong Penggemukan: BK 10-14 kg/hari, PK 12-14%, TDN 65-72%, Ca 0.4-0.6%, P 0.3-0.4%
•	Sapi Potong Induk Bunting: BK 8-10 kg/hari, PK 10-12%, TDN 55-60%, Ca 0.4-0.5%, P 0.3-0.35%
•	Domba Penggemukan: BK 1.2-1.8 kg/hari, PK 14-16%, TDN 65-70%, Ca 0.4-0.6%, P 0.3-0.4%
•	Domba Induk Bunting: BK 1.0-1.5 kg/hari, PK 12-14%, TDN 58-62%, Ca 0.5-0.7%, P 0.35-0.45%
•	Kambing Penggemukan: BK 1.0-1.5 kg/hari, PK 14-16%, TDN 65-70%, Ca 0.4-0.6%, P 0.3-0.4%
•	Kambing Perah: BK 1.5-2.0 kg/hari, PK 16-18%, TDN 65-68%, Ca 0.6-0.8%, P 0.4-0.5%
3.7 Penyesuaian Stok (stock_adjustments)
Untuk stok rusak, kadaluwarsa, atau koreksi.
Trigger (quantity_change negatif):
1.	Update inventaris: qty -, total_cost -
2.	Insert financial_transactions: expense non_cash kategori stock_loss

4.	MANAJEMEN KEUANGAN TERINTEGRASI
4.1 General Ledger (financial_transactions)
Semua pergerakan uang dicatat otomatis dari setiap modul operasional.
Dimensi Transaksi: type (income/expense), cash_flow (cash_in/cash_out/non_cash), category, amount, source_table, source_id, animal_id.
4.2 Mapping Sumber Transaksi (14 Sumber):
1.	Pembelian Pakan - feed_purchases - category: feed_purchase - cash_out
2.	Pemakaian Pakan - feed_consumption - category: feed_usage - non_cash
3.	Pembelian Obat - medicine_purchases - category: medicine_purchase - cash_out
4.	Pemakaian Obat - medicine_usages - category: medicine_usage - non_cash
5.	Biaya Kesehatan (Jasa Vet) - health_records - category: vet_service - cash_out
6.	Biaya Vaksinasi - vaccinations - category: vaccination - cash_out
7.	Biaya Reproduksi - breeding_events - category: breeding - cash_out
8.	Penjualan Ternak - animal_sales - category: animal_sale - cash_in
9.	Pembelian Ternak - animal_purchases - category: animal_purchase - cash_out
10.	Penjualan Susu - product_sales - category: product_sale - cash_in
11.	Penjualan Wol - product_sales - category: product_sale - cash_in
12.	Biaya Tenaga Kerja - labor_expenses - category: labor - cash_out
13.	Biaya Operasional - operational_expenses - category: opex_[category] - cash_out
14.	Penyesuaian Stok - stock_adjustments - category: stock_loss - non_cash
4.3 Laporan yang Dihasilkan:
•	Laba Rugi: Pendapatan - (Biaya cash + Biaya non-cash) dari financial_transactions
•	Arus Kas: Cash in - Cash out (hanya cash_in/cash_out)
•	Biaya per Ekor: Total biaya / jumlah ternak
•	Biaya per Liter Susu: Total biaya sapi perah / total produksi susu
•	Nilai Inventaris: Total nilai stok pakan + obat

5.	MANAJEMEN PRODUKSI HARIAN
5.1 Pencatatan Produksi (daily_production):
•	Sapi Perah: Susu (liter) - per individu atau grup
•	Domba: Wol (kg) - per individu
•	Semua: Daging/bobot potong tercatat di weight_records
5.2 Penggunaan Data Produksi:
•	Dashboard produktivitas harian
•	Grafik tren produksi 7/30/365 hari
•	Perhitungan efisiensi: susu per ekor per hari, susu per kg pakan
•	Benchmarking performa
________________________________________


6.	MANAJEMEN TUGAS & NOTIFIKASI
6.1 Manajemen Tugas (tasks):
Atribut: title, description, assigned_to, due_date, priority (low/medium/high/urgent), status (pending/in_progress/completed/cancelled), related_animal_id, related_herd_group_id.
Auto-generate Tasks:
•	Vaksinasi terjadwal: "Vaksinasi [vaksin] untuk [tag_id]"
•	Birahi terdeteksi: "Inseminasi [tag_id] dalam 12 jam"
•	Perkiraan lahir: "Persiapan kelahiran [tag_id]"
•	Stok menipis: "Order pakan [nama_pakan]"
6.2 Peringatan Dini (alerts):
Tipe Alert: vaccination_due, low_stock_feed, low_stock_medicine, health_issue, breeding_due, task_overdue, weight_loss.
Severity: info, warning, critical.
Trigger Auto-Alert:
•	vaccinations.next_due_date <= hari ini + 3: vaccination_due (warning)
•	feed_inventory.qty < min_threshold: low_stock_feed (critical)
•	medicine_inventory.qty < min_threshold: low_stock_medicine (critical)
•	health_records unresolved + type 'illness': health_issue (warning)
•	breeding_events.expected_due_date <= hari ini + 7: breeding_due (info)
•	tasks.due_date < hari ini + status != 'completed': task_overdue (warning)
________________________________________


7.	MANAJEMEN AKSES & PENGGUNA
7.1 Kontrol Berbasis Peran (RBAC):
•	Owner: Semua akses: lihat, tambah, edit, hapus, laporan keuangan, ekspor data
•	Manager: Semua akses operasional: ternak, kesehatan, pakan, produksi, tugas; Laporan keuangan terbatas
•	Worker: Input data: pakan, produksi, kesehatan dasar; Lihat tugas; Tidak bisa akses keuangan
7.2 Mode Offline:
•	Pekerja dapat mencatat data di kandang tanpa internet
•	Data disimpan lokal (SQLite/IndexedDB)
•	Sinkronisasi otomatis saat koneksi tersedia
•	Konflik resolusi: timestamp terbaru menang

ATURAN BISNIS KUNCI
1.	Moving Average Cost: new_avg = ((qty_lama x avg_lama) + (qty_beli x harga_beli)) / (qty_lama + qty_beli)
2.	Lama Kebuntingan: Sapi = 280 hari, Domba = 147 hari, Kambing = 150 hari
3.	Auto-status Sick: Saat health_record type 'illness' ditambahkan
4.	Auto-status Healthy: Saat semua health_record unresolved diselesaikan
5.	Auto-status Pregnant: Saat pregnancy_check positif
6.	Auto-status Lactating: Saat birth
7.	Auto-status Dry: Saat dry_off
8.	Auto-status Sold: Saat animal_sales
9.	Stok Tidak Boleh Negatif: Constraint CHECK pada feed_inventory.quantity_on_hand >= 0
10.	Alert Stok Menipis: quantity_on_hand < min_threshold
11.	Alert Vaksinasi: 3 hari sebelum next_due_date
12.	Alert Kelahiran: 7 hari sebelum expected_due_date
13.	Composite UNIQUE (source_table, source_id) di financial_transactions mencegah duplikasi
